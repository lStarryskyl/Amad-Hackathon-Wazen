# AGENTS.md — Wazen (وازن) Hackathon Repo

## Quick Reference

**Project:** Wazen — AI personal finance app (4 pillars: Protect, Master Past, Master Present, Master Future)
**Stack:** pnpm workspace (catalog versions in `pnpm-workspace.yaml`), Node 24, Express 5, Expo 54, React Native, PostgreSQL + Drizzle ORM, Clerk v3, OpenAI-compatible AI
**Auth:** Clerk v3 with proxy middleware (mounts BEFORE `express.json()`), Expo token cache
**Codegen:** OpenAPI 3.1 → orval → Zod schemas + React Query hooks
**ESM only:** all packages `"type": "module"`, Node `--enable-source-maps`

---

## Essential Commands

```bash
pnpm install                         # Install (run once)
pnpm run typecheck:libs               # Build shared libs (required before first run)
pnpm run typecheck                    # Typecheck everything
pnpm run build                        # Build all packages

# Database
pnpm --filter @workspace/db run push  # Apply Drizzle migrations

# Codegen (after schema changes — DO NOT skip)
pnpm codegen                          # Regenerates api-client-react + api-zod from OpenAPI spec

# Dev servers (need 3 terminals)
pnpm --filter @workspace/api-server run dev      # API on :8080 (builds first — NOT hot reload)
pnpm --filter @workspace/wazen-landing run dev   # Landing on :3000
pnpm --filter @workspace/finance-mobile run dev  # Expo DevTools
```

## API Server Architecture

### Route Registration Order (`artifacts/api-server/src/routes/index.ts`)

Routes are mounted on `/api` in this order — add new routes to the correct position:
1. `health` — `/healthz` (no auth)
2. `profile` — `/profile`
3. `accounts` — `/accounts`
4. `transactions` — `/transactions`
5. `categories` — `/categories`
6. `goals` — `/goals`
7. `consent` — `/consent`
8. `onboarding` — `/onboarding`
9. `summary` — `/summary`
10. `ai` — `/ai/*`
11. `intelligence` — `/ai/regret-score`, `/ai/rescue-plan`, `/ai/money-story`
12. `simulations` — `/simulations`
13. `engagement` — `/guardrails`, `/streaks`, `/achievements`, `/checkins`, `/alerts`, `/patterns`
14. `devReset` — `/dev/reset` (dev only)

### Middleware Mounting Order (critical — `artifacts/api-server/src/app.ts`)

1. `pino-http` (logging)
2. **Clerk proxy** (`/__clerk`) — must be BEFORE `express.json()`
3. `cors`
4. `express.json()`
5. `express.urlencoded()`
6. `clerkMiddleware` (Clerk session parsing — uses `publishableKeyFromHost` via proxy)
7. Routes at `/api`
8. Global error handler (returns `InternalError` JSON)

### Middleware (`artifacts/api-server/src/middlewares/`)
- `requireAuth.ts` — sets `req.userId` from Clerk session
- `requireConsent.ts` — checks user has consented to data processing
- `clerkProxyMiddleware.ts` — streams raw Clerk auth bytes

### Lib (`artifacts/api-server/src/lib/`)
- `aiOrchestration.ts` — OpenAI client factory; tries user's encrypted key first, falls back to server `AI_API_KEY`; has `generateRescueNarrative()`, `generateMoneyStory()`
- `financialIntelligence.ts` — computeRegretScore, buildRescueActions, buildMoneyStoryContext (SQL logic, no AI calls)
- `fallbackNarratives.ts` — local fallback narratives when OpenAI is unreachable
- `patternDetection.ts` — detectBehavioralPatterns from transaction data
- `simulationEngine.ts` — run hypothetical financial scenarios
- `encryption.ts` — decryptApiKey for user-provided OpenAI keys
- `pushNotifications.ts` — Expo push notifications
- `userProvisioning.ts` — getOrCreateUser (ensure DB row exists for Clerk user)
- `logger.ts` — pino logger instance

## Codegen Workflow (Important — CI Checks for Drift)

1. Modify Drizzle schema in `lib/db/src/schema/*.ts`
2. Run `pnpm --filter @workspace/db run push` to apply to DB
3. Update OpenAPI spec in `lib/api-spec/openapi.yaml` — this is the source of truth
4. Run `pnpm codegen` — triggers orval from `lib/api-spec/orval.config.ts` which generates:
   - `lib/api-zod/src/generated/` (Zod schemas + types)
   - `lib/api-client-react/src/generated/` (React Query hooks + schemas)
5. Commit ALL generated files — CI runs `git status --porcelain` on generated dirs and fails if dirty

**Important:** `lib/api-client-react/src/index.ts` has explicit manual re-exports (intelligence, simulations, engagement) that come BEFORE the wildcard `export * from "./generated/api"`. Manual exports win over generated ones. Add new manual exports in the same pattern.

**Important:** `lib/api-client-react/src/custom-fetch.ts` provides the HTTP client — uses `setBaseUrl()` and `setAuthTokenGetter()` for per-request auth. The token getter is wired in `app/_layout.tsx` and `(home)/_layout.tsx`.

## Bold Design System

**Layout:** Spacing scale 4/8/12/16/20/24/32/40/48/64, border radius 8/12/16/20/24/32/9999
**Touch Target Minimum:** 44×44pt

**Light Palette Key Tokens:**
| Token | Value | Token | Value |
|-------|-------|-------|-------|
| Primary | `#0077BC` | Secondary | `#009866` |
| Success | `#16A34A` | Warning | `#D97706` |
| Danger | `#DC2626` | Surface | `#FFFFFF` |
| Surface Elevated | `#F3F4F6` | Text | `#111827` |
| Border | `#E5E7EB` | Muted | `#9CA3AF` |

**Dark Palette:** auto-switches via `BoldProvider` — persists in AsyncStorage key `wazen:theme`, respects system preference on first load.

**Deprecated:** `hooks/useColors.ts` — DO NOT use, DO NOT import. Use `useBoldColors()` from `@/constants/themes`.

- `achievements` — user achievements/badges

## Key Quirks

- **Fonts are `@expo-google-fonts/archivo` + `@expo-google-fonts/jetbrains-mono`**, loaded in `app/_layout.tsx` with `useFonts`. Not "Archivo Black" as a single package.
- **API `dev` builds first** — `dev = build && start` in package.json, not hot reload. Edit → rebuild.
- **BoldText uses fontFamily "ArchivoBlack"** (the specific font face name, not the package name).
- **Expo Router file-based nav**: `app/(auth)/sign-in.tsx`, `app/(home)/(tabs)/index.tsx`, etc.
- **Clerk proxy MUST mount before express.json()** — middleware order matters in `app.ts`.
- **`@workspace/api-client-react` manual exports win over generated** — `src/index.ts` lists explicit re-exports before `export * from "./generated/api"`.
- **Regret Score terminology** — use "Regret Score" everywhere, not "Regret Meter" or "Regret Score™".
- **Bundle IDs**: iOS `com.wazen.finance`, Android `com.wazen.finance`, scheme `wazen://`.
- **AI model-agnostic**: uses `AI_BASE_URL` + `AI_MODEL` env vars; works with OpenAI, Groq, Together, etc.
- **User can bring their own OpenAI key** — encrypted in `users.encryptedOpenAiKey`, decrypted via `encryption.ts`. Falls back to server `AI_API_KEY`.
- **pnpm catalog versions** in `pnpm-workspace.yaml` — update there, not per-package.
- **esbuild banner** in `build.mjs` patches `globalThis.require` + `__dirname`/`__filename` for ESM compatibility with CJS packages.
- **Light mode default** — `BoldProvider` persists via AsyncStorage key `wazen:theme`, reads system preference on first mount.
- **TypeScript base config** in root `tsconfig.base.json` — individual packages extend it.
- **`mockup-sandbox/`** exists under `artifacts/` — a standalone Expo testbed, not part of main apps.
- **`useColors` is deprecated** — zero references remain. Never import it. Use `useBoldColors()`.
